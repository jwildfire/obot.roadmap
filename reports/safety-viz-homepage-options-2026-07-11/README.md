# safety.viz homepage — five layout directions

Design-options report for @jwildfire's 2026-07-11 feedback on the safety.viz v1.0
homepage ("too much text — a 1–2 sentence intro; the rest to About"), tracked as
[safety.viz#29](https://github.com/jwildfire/safety.viz/issues/29) with the baseline
copy cut shipped as draft [PR #30](https://github.com/jwildfire/safety.viz/pull/30).

Five layout theses (A baseline · B chart-forward split hero · C live-demo-first ·
D gallery-first · E keynote editorial), each rendered as live HTML in the site's own
keynote theme with the renderers' real evidence-baseline screenshots (downscaled)
as thumbnails. Ends with a recommendation (D, runner-up B) and the next step.

**How it was generated:** hand-built by the 😺🤖 07-11 sv-v1 session from the
safety.viz v1.0 site generator source (`scripts/site-lib.mjs` @ dev), the site
theme tokens, and `site/config.json` renderer metadata. Mockups are illustrative,
not pixel-final builds.
