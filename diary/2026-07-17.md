## Work completed

### 🔬 Five ideas → a grounded feasibility assessment ([hub report](https://jwildfire.github.io/obot.roadmap/reports/safety-graphics-improvement-assessment-2026-07-17/))

- Jim Buchanan's five ideas — a hep-explorer composite plot for baseline-abnormal DILI, finishing the Renal explorer, a QT explorer, benefit-risk tools (forest plot + value tree), and recurrent-AE analysis — were broken into actionable requirements each with a high-level feasibility read (Hep = High, Renal = Moderate, QT = Moderate, BR = Exploratory/gated, Recurrent-AE = Exploratory with quick wins).
- Grounded against the actual sources as they arrived: the Tesfaldet/Hayashi *Drug Safety* 2024 composite-plot paper (with the FDA public-domain [reference code](https://github.com/FDA/Composite-eDISH-Plot)), the ISG Renal Safety Explorer manual, and the CSRC QT tool mockup + draft clinical workflow. One earlier C-QTc speculation was corrected against the reviewed mockup. Published to the hub (report index + changelog 1.7.5).

### ❤️ QT Safety Explorer, Phase 1 — the first cardiac-safety renderer ([sv#68](https://github.com/jwildfire/safety.viz/issues/68) → [PR #70](https://github.com/jwildfire/safety.viz/pull/70); parent [hub #36](https://github.com/jwildfire/obot.roadmap/issues/36); ⚡️🤖 qt-explorer-p1)

- One renderer, three coordinated views over a shared correction (QTcF / QTcB / Heart Rate): a **central-tendency** view (Δ / placebo-corrected ΔΔ, two-sided 90% CI band, peak-visit marker, and the **ICH-E14 metric**), the flagship **outlier scatter** (450/480/500 ms absolute diagonals + 30/60 ms change lines, per-arm shapes), and a **categorical** by-arm exceedance table. Ported from SafetyGraphics/qtexplorer per the CSRC mockup.
- Real demo data: `site/data/adeg.csv` projected from pharmaverseadam CDISC Pilot ADEG — QTcF/QTcB/HR across 254 participants and three arms. Ships marked **Experimental** (badge on the page title); QTcI, PR/QRS/JT, drill-down, hysteresis, and a moxifloxacin arm are Phase 2. A clinical guide page ports the CSRC step workflow.
- A 3-lens design critique before coding and a 9-finding adversarial correctness review after (all fixed, +3 regression tests). 50 new unit tests, 9 new e2e.

### 🧪 hep-explorer composite plot — DILI in abnormal-baseline populations ([sv#67](https://github.com/jwildfire/safety.viz/issues/67) → [PR #69](https://github.com/jwildfire/safety.viz/pull/69); ⚡️🤖 hep-composite)

- A new **View** on the shipped hep-explorer for the population the standard ×ULN eDISH both false-alarms on and masks change in: it anchors the assessment to each subject's *own* baseline (×BLN) — two eDISH scatters that keep baseline-quadrant colors to show migration, a four-panel ×BLN shift plot, a color-coded migration table, and a by-arm concern/benefit summary. FDA-faithful algorithm ported from the reference code.
- A synthetic chronic-liver-disease demo cohort added; cross-panel participant tracing (hover/click traces a subject across all six panels). 22 requirement-keyed unit tests + e2e; a 6-finding adversarial review, all fixed. Held open for **Jim's review** before merging — the review came back and it merged overnight.

### 🧭 Site navigation — Gallery dropdown + header icons ([sv#71](https://github.com/jwildfire/safety.viz/issues/71) → [PR #72](https://github.com/jwildfire/safety.viz/pull/72); [sv#73](https://github.com/jwildfire/safety.viz/issues/73) → [PR #74](https://github.com/jwildfire/safety.viz/pull/74); 👯🤖 nav-gallery-dropdown, gh-icon-headers)

- The top-nav **Gallery** link now expands into a **data-driven dropdown** of every chart (click Gallery for the index, or a chart to land on its page) — sourced from the renderer registry, so new renderers appear automatically. Accessible disclosure pattern, hover + click + keyboard.
- **Header words → icons**: the repository link is now the GitHub mark icon and the hub link the obot brand mark, with accessible names preserved (`aria-label` + SVG `<title>`, `currentColor`). The same GitHub-icon swap shipped to the obot.roadmap hub header (direct to main). An inventory confirmed the pkgdown/SVG sites (gsm.safety, open.gismo) already used icons — no change needed.
