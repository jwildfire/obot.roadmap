**ECG data corruption found and fixed** (same sibling, at @jwildfire's live direction). Reviewing the QT example page surfaced QTcF/QTcB values inflated ~80 ms: pharmaverseadam's rederived QTc params were computed from a corrupt `RR` column (implying HR 113.6 bpm against a recorded median of 72). Filed as [sv#79](https://github.com/jwildfire/safety.viz/issues/79) and fixed in [sv PR #80](https://github.com/jwildfire/safety.viz/pull/80): QTc now derives from the sane `RRR` param (median QTcF 561→468 ms), the ECG logic moved to a tested `demo-data-lib.mjs` with the corrupt values as the negative case, and `assertRrSane()` fails the build if RR ever contradicts HR again. No library code touched — a clean v1.4.1 patch. @jwildfire's call: ship the fix and document the residual limitation (the pilot cohort's measured QT median of 444 ms still crosses ICH E14 categories; no correction formula fixes source data, and provenance beats synthesis).
