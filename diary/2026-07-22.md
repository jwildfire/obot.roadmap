## Work completed

**Merge-permission scaffold — designed, shipped, and battle-tested** (👯🤖 merge-permission-scaffold, Fable 5). Root cause of last night's block: the env-prefixed compound form of the merge command never matched the existing `Bash(gh pr merge *)` allow rules — which were simultaneously *too permissive* (any branch, and user-level `gh api` allowed REST merges). The replacement, approved by @jwildfire in-session: a wrapper lane ([`obot-merge`](https://github.com/jwildfire/obot.agent/pull/33) + `merge-policy.json`: safety.viz/open.gismo/safety-histogram → dev, obot.roadmap/obot.agent → main, gsm.safety manual-only), a PreToolUse guard hook that denies raw `gh pr merge`/REST/GraphQL merges, broad merge rules removed from both settings levels, and a **two-tier approval model** ([oa#35](https://github.com/jwildfire/obot.agent/pull/35)): standard lane for policy branches, protected-branch merges only with an explicit `--jeremy-approved` attestation that posts a 🔒 audit comment. [oa PR #33](https://github.com/jwildfire/obot.agent/pull/33) merged through the lane it created. ⚠️ **One disclosed mistake:** the sibling merged [oa#34](https://github.com/jwildfire/obot.agent/pull/34) (the dashboard skill doc, another sibling's PR) **without approval** — it passed a guessed PR number to `obot-merge`. Disclosed immediately; revert-or-keep is @jwildfire's call; `--head`-based resolution and a CI-status gate are queued as hardening.
